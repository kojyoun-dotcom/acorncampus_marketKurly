
### 1. 깃 협업 규칙 (Git Workflow)

초보자들에게 가장 무서운 건 내 코드가 날아가거나, 남의 코드를 망가뜨리는 것입니다.

* **Main 브랜치는 금지:** "Main 브랜치에 직접 Push하지 마세요."라고 엄격히 말해주세요.
* **1인 1브랜치:** `feature/header`, `feature/banner` 처럼 자기 담당 영역 이름으로 브랜치를 파서 작업하게 하세요.
* **PR(Pull Request) 후 합치기:** 작업이 끝나면 팀장(본인)에게 PR을 날리게 하고, 팀장이 확인 후 합치는 구조로 가야 코드가 꼬이지 않습니다.

### 2. CSS 충돌 방지권 (Class Naming)

각자 작업한 CSS를 합쳤을 때, 서로 같은 클래스명(`.container`, `.button`)을 써서 레이아웃이 터지는 경우가 99%입니다.

* **접두어(Prefix) 사용:** 본인 담당 영역의 약자를 클래스명 앞에 붙이게 하세요.
* 헤더 담당: `.hd-login`, `.hd-search`
* 상품 리스트 담당: `.prod-card`, `.prod-title`


* **박스 모델 고정:** 공통 CSS에 아래 코드를 넣고 시작하겠다고 선언하세요. (이거 안 하면 사이즈 계산하다 밤샙니다.)
```css
* { box-sizing: border-box; }

```

### 3. 마켓컬리 '브랜드 컬러' 통일

팀원들이 각자 눈대중으로 보라색을 고르면 사이트가 알록달록해집니다. `common.css`를 만들어서 공유하세요.

* **메인 보라색:** `#5f0080`
* **서브 보라색:** `#7209b7`
* **포인트 회색:** `#333333` (글자색), `#666666` (설명글)
* **기본 폰트:** 가능하면 'Noto Sans KR'로 통일하자고 가이드를 주세요.

### 4. 3일의 전술: "완성보다 연결"

입문자들은 한 영역의 디테일에 집착하다 시간을 다 쓰곤 합니다.

* **1일차 목표:** "모양 안 예뻐도 좋으니 HTML 태그 다 짜고 브라우저에 내용만 나오게 하기"
* **이미지 확보:** 마켓컬리 사이트에서 `F12` 눌러서 이미지 주소를 따오거나, 아예 팀장이 공용 폴더에 이미지를 몇 개 넣어주고 시작하는 게 빠릅니다.
